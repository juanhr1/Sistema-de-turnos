🪜 5. Análisis de Fallos

Servicio de Turnos
No se pueden generar ni gestionar turnos. Se pierde la funcionalidad principal.
Debe continuar parcialmente mientras se recupera el servicio.
* Alta disponibilidad : En lugar de tener un solo Servicio de Turnos, se pueden tener varias versiones funcionando al mismo tiempo. Si una se cae, otra sigue trabajando.
* Balanceador de carga: Organizador que reparte las solicitudes entre varias copias del servicio. Así no se sobrecarga una sola y se reduce el riesgo de falla.
* Replicación de base de datos: La información de los turnos puede guardarse en más de un lugar. Si una base de datos falla, otra puede seguir funcionando sin perder datos.
* Monitoreo constante: Usar herramientas que den aviso inmediato cuando algo empieza a fallar, para poder actuar rápido.
* Circuit breaker: Si un servicio no está respondiendo, el sistema deja de intentar comunicarse con él por un momento. Esto evita que todo el sistema se bloquee mientras el servicio se recupera.

🪜 6. Service Discovery

Los demás servicios no podrían localizarlo y fallaría la comunicación.
Consultando un servidor de registro dinámico (Service Registry).
Permite encontrar servicios automáticamente sin depender de IP fija.
Entre todos los microservicios: Usuarios, Turnos, Notificaciones y Administración.
