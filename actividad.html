🪜 2. REST o Eventos

Interacción REST Evento ¿Por qué?
1: Solicitud de turno ✌️ El cliente hace una petición directa y necesita respuesta inmediata.
2: LLamar turno ✌️ El empleado necesita confirmación inmediata para continuar.
3: Notificación ✌️ Se genera automáticamente cuando cambia el estado del turno.

🪜 3. Síncrono o Asíncrono

Interacción Síncrono Asíncrono Justificación
1: Solicitud de turno 💖 El cliente puede esperar su número de turno.
2: LLamar turno 💖 El empleado necesita respuesta inmediata.
3: Notificación 💖 Puede ejecutarse sin bloquear el sistema principal.

🪜 4. Eventos Importantes del Sistema

1️⃣ Turno generado

2️⃣ Turno llamado

3️⃣ Turno atendido

4️⃣ Turno cancelado

5️⃣ Usuario autenticado

6️⃣ Falla detectada en un servicio

Estos eventos permiten que otros servicios reaccionen sin depender directamente entre sí.

Dibujen:

💡 Ayuda rápida
REST → petición directa

Service Discovery → permite encontrar servicios automáticamente
